# inbox - 用户手工塞重大事件

把你想让 stock-helper 知道的"重大事件 / 突发新闻 / 法规变化 / 个人观察"用 markdown 文件放进这个目录。

每次运行 `bash scripts/refresh.sh` 时，`distill_recent.py` 会读取所有 `*.md` 并喂给 LLM，让它整合进 `recent.md`。

## 文件命名建议

`YYYY-MM-DD-短标题.md`，例如：

- `2026-05-22-互联网券商监管.md`
- `2026-05-24-霍尔木兹谈判.md`

## 文件内容模板

```markdown
# 简短标题

date: YYYY-MM-DD
type: 事件 / 政策 / 个股 / 行业

正文：要点 1～3 句话。

影响：哪些标的、哪些观点会被改变。
```

## 何时塞文件

- 看到重要新闻而当天财经门户抓不到
- 想覆盖 KB 里某个明显过时的旧观点
- 任何你希望 stock-helper 在下次回答时纳入考虑的东西

## 处理后

inbox 里的文件**不会被自动删除**。如果你想清掉旧条目，手工 `rm` 即可。
