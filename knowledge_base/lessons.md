# 复盘教训库（持续追加）

> 每次推荐 → 次日收盘对照 → 失误归因，沉淀为评分/Prompt 改进依据。
> daily_advice 在生成新清单前**必须**读这份。

## 2026-05-29（针对 2026-05-30 清单的复盘）

### 失误 1：把"昨日大涨"等同于"今日龙头"

**事实**：北京君正(昨升今 **-6.40%**)、中芯国际(**-6.10%**) — 5/28 score_universe 给高分时正是它们昨日大涨，次日均值回归直接吃 5-6 个点。

**根因**：`score_universe.py` 的主线龙头评分把"涨 4-9%"列为满分项，把"今日成交额排名"作为加分，但未对"连涨 2-3 日"的疲态做扣分。教授 [基于 2026-05-26] 多次警示"半导体破防/暴锤"恰是这个场景。

**改进**：
- score_universe 增加"连涨疲劳因子"：如果该股近 3 日累计涨幅 > 15%，主线评分 ×0.6
- prompt 强制要求 LLM 检查"次日均值回归概率"，对追涨型主线必须标注 "Day-2 风险"
- 买入区间下限不应高于昨日实际最低价，避免"区间定在现价以下却用昨日收盘做基准"

### 失误 2：价值股买入价过于保守，震荡上行市空仓

**事实**：风华高科(+9.02%) / 建行(+0.59%) / 长江电力(+2.10%) / 茅台(+3.92%) — 4 只全部收涨，因买入区间普遍 -3~-5% 全部未触发。账户错过整段上涨。

**根因**：LLM 默认"价值股要等深回调"，但在主线发酵期，价值股会同步水涨船高。买入区间应该是 **现价 -1% ~ 现价**（追涨容忍）+ **底部 -5% 加仓位**（分层），而不是只设一个"等回调"区间。

**改进**：
- 价值股 prompt 改为"双档买入"：第一档 现价±1% 起建仓 50%，第二档 -5% 加仓 50%
- 不要把"价值股"和"等深回调"绑死——市场强势时这是空仓陷阱

### 失误 3：自相矛盾——风华涨停日推荐价低于昨日收盘

**事实**：风华高科 5/28 收 48.64（评分入选），daily_advice 给"买入价 48.5-50.5"，但今日开盘即 49.55 直冲涨停 53.03 — 推荐区间逻辑断裂。

**根因**：LLM 看到"现价 51.04"（实时盘中）就以为"低一点 48-50 能买到"，没有意识到 51 是连续上攻中的报价，再回踩 48 等于让大涨被反转。

**改进**：
- prompt 加约束：买入下限不得低于"昨收 -3%"且不得低于"今日最低价 -1%"
- 对连涨/涨停标的，应该直接归入"暂不追"或"突破新高买入"两类，不要给"逢回调买"
